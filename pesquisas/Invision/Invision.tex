\section{Introdução}%GGN

É imprescritível que o planejamento é parte necessária a concepção de uma aplicação ou site, e ter um layout navegável pode tornar a ideia mais clara para os envolvidos, e também é uma forma de rever os conceitos assim recebendo um retorno, mesmo que em estágio precoce, além de melhorar a usabilidade em um estágio inicial e corrigir erros.

\section{O que é?}%GGN

O invision é uma ferramenta que tem a possibilidade de fazer o projeto ganhar vida, sair de um modo parado e tornar-se dinâmico. Após o projeto concluído, envia-se este para o Invision e adiciona algumas transições e animações e o protótipo passa a ser clicável.


Através do build mode você pode selecionar hotspots (lugares marcados onde o usuário irá clicar na tela, Ex: botões de logar, cadastrar, voltar) e marcar qual será a próxima tela que aparecerá após está ação. De forma mais rápida você pode criar Templates, para itens recorrentes com menus, e aplicar de forma global nos seus protótipos.

Os usuários que estão dentro do projeto podem clicar em qualquer lugar da tela, comentar nos protótipos, responder a comentários dos outros stakeholders, por isso é muito indicado quando se quer colaboração e feedback com os clientes.

\section{Vantagens}%GGN

\begin{itemize}
\item Caso o protótipo seja de um dispositivo móvel ele fornecerá um link através de um SMS ou e-mail para abrir no celular.
\end{itemize}

\begin{itemize}
\item É de fácil utilização, sem necessidade de nenhum conhecimento, em poucos minutos o desenvolvedor navega pela plataforma com tranquilidade.

\end{itemize}

\begin{itemize}
\item A Ferramenta é colaborativa, ou seja, várias pessoas podem trabalhar no mesmo protótipo ao mesmo tempo.
\end{itemize}

\begin{itemize}
\item É uma Ferramenta de desenho de protótipos com mãos livres, o que possibilita criar um primeiro esboço e testar usabilidade sem dificuldades.
\end{itemize}

\begin{itemize}
\item Possibilidade de testar workflow das páginas.
\end{itemize}

\begin{itemize}
\item Possui um sistema de gerenciamento de projeto parecido com o github.
\end{itemize}

\section{Desvantagens}%GGN

\begin{itemize}
\item Possibilidade de desenvolver protótipos mais elaborados somente utilizando duas ferramentas pagas, que no caso são o photoshop e o sketch.
\end{itemize}

\begin{itemize}
\item Na ferramenta não é possível desenvolver protótipos, apenas importar as imagens já prontas.
\end{itemize}

\begin{itemize}
\item A Versão com mais funcionalidades é paga, valores variam de 0 a 90 dólares.
\end{itemize}

\begin{itemize}
\item A ferramenta só permite um projeto na versão gratuita.
\end{itemize}
